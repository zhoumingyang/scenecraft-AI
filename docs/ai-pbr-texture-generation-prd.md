# AI PBR Texture Generation PRD

## Summary

AI PBR texture generation lets a user select a mesh or the ground plane, describe a material, and generate a single 3x2 PBR texture atlas with OpenRouter `openai/gpt-5.4-image-2`. The generated atlas is immediately applied to the selected target by reusing the existing mesh material texture fields.

This feature does not introduce a new material schema. It stores the same atlas URL in the existing `TextureSchema` values for `diffuseMap`, `metalnessMap`, `roughnessMap`, `normalMap`, `aoMap`, and `emissiveMap`, with different UV offsets and a shared repeat value.

## Goals

- Generate realistic PBR material atlases from a short user prompt.
- Apply the generated atlas directly to the currently selected mesh or ground plane.
- Keep the property panel focused on the selected mesh or ground object.
- Show configured texture thumbnails in each material texture row.
- Save generated atlases into the project AI asset library for reuse.
- Preserve existing AI image, AI 3D, manual texture upload, AI asset, and Poly Haven material flows.

## Non-Goals

- Splitting the atlas into six separate image files.
- Changing `TextureSchema`, `ResolvedTextureSchema`, or the main project snapshot shape.
- Supporting imported model internals with multiple nested materials in the first version.
- Adding OpenAI direct API support or a GPT-5.5 planning step in the first version.

## User Flow

1. User selects a mesh, or selects the ground helper while it is in `plane` mode.
2. The Appearance section shows an "AI PBR Texture" action.
3. User opens the generation dialog, enters a prompt such as "realistic rock texture" or "rusted metal".
4. The app sends the prompt to `/api/ai/textures/pbr/generate`.
5. The server calls OpenRouter `openai/gpt-5.4-image-2` with a fixed 3x2 PBR atlas prompt.
6. The returned atlas URL is immediately applied to the current target.
7. The atlas is recorded as a pending AI generation with PBR atlas metadata.
8. The material rows refresh in place and each map row shows a thumbnail.
9. Opening any texture Configure panel shows the same atlas image with that map's UV transform.
10. Saving the project uploads the atlas through the existing AI asset pipeline and rewrites texture URLs to Blob URLs.

## UX Requirements

- The property panel must stay on the mesh or ground appearance UI throughout the flow.
- Generation uses a lightweight dialog launched from `MeshAppearanceSection`.
- The dialog includes prompt input, generate button, loading state, and error messaging.
- On success, the atlas is applied automatically without a second Apply click.
- `TextureConfigRow` shows a small preview thumbnail whenever `texture.url` exists.
- AI asset library entries generated by this feature display a PBR atlas marker and can be applied as a full atlas.
- Plain AI images without PBR atlas metadata continue to behave as single texture images.

## Atlas Layout

The generated atlas is a 3-column by 2-row image:

| Slot | Field |
| --- | --- |
| Top left | `diffuseMap` |
| Top center | `metalnessMap` |
| Top right | `roughnessMap` |
| Bottom left | `normalMap` |
| Bottom center | `aoMap` |
| Bottom right | `emissiveMap` |

Each texture field uses:

- `url`: generated atlas URL
- `assetId`: uploaded asset ID after save, empty before upload
- `externalSource`: `null`
- `repeat`: `[1 / 3, 1 / 2]`
- `rotation`: `0`

Offsets:

- `diffuseMap`: `[0 / 3, 1 / 2]`
- `metalnessMap`: `[1 / 3, 1 / 2]`
- `roughnessMap`: `[2 / 3, 1 / 2]`
- `normalMap`: `[0 / 3, 0 / 2]`
- `aoMap`: `[1 / 3, 0 / 2]`
- `emissiveMap`: `[2 / 3, 0 / 2]`

## API Contract

Endpoint: `POST /api/ai/textures/pbr/generate`

Request:

```ts
{
  prompt: string;
  imageSize?: ImageGenerationImageSize;
  seed?: number;
  targetKind?: "mesh" | "ground";
  targetId?: string;
}
```

Response:

```ts
{
  atlasImageUrl: string;
  model: "openai/gpt-5.4-image-2";
  prompt: string;
  seed: number | null;
  traceId: string | null;
  layoutVersion: 1;
}
```

The route requires an authenticated session and `OPENROUTER_API_KEY`.

## Prompt Requirements

The server prompt must ask for:

- one single 3x2 atlas image
- no text, labels, captions, watermarks, borders, gutters, or annotations
- strict slot order matching the atlas layout
- realistic physically based material maps
- seamless/tileable material content
- square output by default
- each slot showing only its corresponding PBR map information

## Persistence

The atlas is stored as a normal AI generated image result, with optional metadata:

```ts
{
  kind: "pbr_texture_atlas";
  atlasLayoutVersion: 1;
  targetKind: "mesh" | "ground";
  targetId?: string;
}
```

This metadata is only for display and reuse behavior in the AI asset library. Rendering and project snapshots continue to rely on the existing material texture schemas.

## Engineering Design

- Add `openai/gpt-5.4-image-2` to the image model registry.
- Add PBR generation request and response schemas in the AI API contracts.
- Add a frontend API wrapper for the new route.
- Add a PBR atlas helper that returns a standard `MeshMaterialPatch`.
- Add a generation dialog that applies the patch on success.
- Extend AI generation/result types with optional metadata.
- Extend the AI asset library to recognize and reapply PBR atlas results.
- Keep all existing AI image and AI 3D composer behavior unchanged.

## Acceptance Criteria

- A selected mesh can receive a generated PBR atlas from one prompt.
- The ground plane can receive a generated PBR atlas from one prompt.
- All six texture rows show a thumbnail after application.
- All six texture Configure dialogs show the same image URL and distinct atlas UV values.
- Generated atlases are saved into the AI asset library and can be reused as a full PBR atlas.
- Existing plain AI image application remains single-map only.
- Project save/load preserves the atlas material references.
- `npm run lint` passes.
